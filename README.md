# 工单管理与图表展示系统

## 一、项目简介

本项目为一个基于 Vue3 开发的简易工单管理系统，主要实现了：

* 登录权限控制
* 工单表格展示
* 管理员删除功能
* ECharts 图表展示
* 表格与图表联动更新

项目整体采用组件化开发方式，实现了基础的前端工程结构与响应式数据联动逻辑。

---

## 二、技术栈

本项目使用的技术如下：

| 技术             | 用途     |
| -------------- | ------ |
| Vue3           | 前端框架   |
| Vite           | 项目构建工具 |
| Ant Design Vue | UI组件库  |
| ECharts        | 图表展示   |
| Vue Router     | 路由管理   |
| JavaScript     | 页面逻辑开发 |

---

## 三、功能实现

### 1. 登录权限控制

* 用户名为 `admin` 时：

  * 拥有管理员权限
  * 可以删除工单

* 其他用户名：

  * 为普通用户
  * 只能查看数据

---

### 2. 工单列表展示

系统使用表格组件展示工单数据，包括：

* ID
* 项目名称
* 是否加班
* 工时
* 创建时间

---

### 3. 删除功能

管理员可删除工单数据。

删除后：

* 表格自动更新
* 图表自动重新统计并刷新

---

### 4. 图表展示

系统使用 ECharts 实现柱状图展示。

图表内容：

* 按项目统计累计工时
* 自动根据当前表格数据更新

## 五、我对AI协作开发的理解

首先，谈一谈完成小项目的过程，我是将该题目的需求文档交给gpt，然后发送“我选择本地 VS Code + Vite了，告诉我具体操作步骤，从0开始”这样的提示词给它，他就这样一步一步的告诉我怎么操作，事实上这个小项目前端开发过程中并没有碰到什么难事。

其次，谈一谈使用AI辅助开发的感想，这个过程中我体会到了gpt的强大，几乎不用动脑，基于我熟练使用vscode的经验很快就完成了，但是AI是需要指令来完成任务的，正是因为需求文档的详细准确，它才会更高效的完成任务，但是所作出的成果是面向人的，自然也就需要人来把关（定义边界，做判断，验结果，兜底），这只是一个简化的前端，如果场景变得更复杂，它出错和跑偏的概率还是会明显上升，因此维护和纠错这部分并没有消失，至少目前看来很难将这些环节彻底交给AI。人如何驾驭AI很大程度上决定了AI的产出质量。直到今天，AI被抬的很高，但我仍然相信着开发工程师切实的价值。

最后，感谢贵公司的测试题，我突然发现找工作这段时间一直在受挫，常常因为面试的失败否定自己，却也没有静下心来去做一件令我有成就感的事情，我很开心碰到你们，也希望与你们这样认真的同事共处。
